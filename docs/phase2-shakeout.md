# Обкатка и дебаг Phase 2 (автосоздание заданий)

План проверки пайплайна автопилота `--autopilot` / `create_task_from_set` на реальной
машине с A-Parser. Цель — довести заполнение формы Task Editor и создание задания под
«набор» до надёжной работы, ничего не сломав в очереди.

## 0. Предусловия

- A-Parser Pro запущен, web-UI доступен (`aparser_ui_url`, `aparser_ui_password` заданы).
- Playwright + браузер установлены на этой машине (`playwright install chromium`).
- В `queries_dir` есть хотя бы один **набор** без задания: подпапка `queries/<группа>/`
  с `.txt`-файлами, для которой нет `results/<группа>/` и нет задания в очереди.
- В A-Parser сохранён **Task preset** с нужным парсером/конфигом (напр. `aparser`).

Ключи конфига для обкатки:
```json
"autopilot_template_task": "aparser",   // имя Task preset (обязательно)
"autopilot_config_preset": "200t",      // опц.; пусто — не трогать
"autopilot_dry_run": true,              // на обкатке ВСЕГДА true
"autopilot_create_tasks": false,        // включим последним
"aparser_root": ""                      // пусто — выведется из exe/queries_dir
```

## 1. Сухой прогон (dry-run), видимый браузер

```
py aparser_monitor_ui.py --autopilot-test --debug
```
Что делает: открывает видимый браузер, находит первый набор без задания, заполняет форму
Task Editor и снимает `data/ui_dumps/autopilot_preview_<набор>.png`. **Задание не
добавляется** (dry-run форсирован внутри `run_autopilot_test`).

### Что проверить на превью-скрине
1. **Открылся Task Editor** (а не остался экран очереди).
2. **Task preset** = ожидаемый (`aparser`), и настройки применились — парсер/потоки те,
   что в пресете (значит `_set_combo` + `LOAD_PRESET_JS` сработали).
3. **Config preset** = `200t` (если задан).
4. **Queries from: File** выбрано (радио), а не Text.
5. **Select File** содержит все файлы набора через запятую, путь относительно корня
   A-Parser (напр. `queries/brave/B0001_….txt, queries/brave/B0002_….txt`).
6. **Results → File name** = `$queriesfile` (унаследовано, не трогаем).

## 2. Типовые сбои и куда смотреть

| Симптом (лог/скрин) | Причина | Что править |
|---|---|---|
| `элемент не найден 'Config preset'/'Task preset'/'Select File'` | другой язык UI или иная подпись | `FIELD_INPUT_JS` матчит по тексту подписи — свериться с реальной подписью в дампе; добавить RU-вариант |
| `радио 'Queries from: File' не найдено` | иная вёрстка радио | `SELECT_FILE_RADIO_JS` (ищет `name="queriesFrom"` + boxLabel «File») |
| Task preset выбран, но настройки не применились | пресет грузится отдельной кнопкой, которую не нашли | `LOAD_PRESET_JS` — уточнить селектор кнопки загрузки рядом с полем |
| В Select File пусто/один файл | комбо-филл перетёрт или путь не резолвится | проверить `_aparser_rel_path` (задать `aparser_root`); проверить, что поле — textfieldmulti |
| Комбо не открывает список | ExtJS ждёт клик по триггеру, не по инпуту | в `_set_combo` кликать trigger `.x-form-trigger`, затем boundlist |
| Скрин пустой/белый | не дождались отрисовки | увеличить `wait_for_timeout` после открытия редактора |

Селекторы-якоря (в `aparser_monitor_ui.py`): `FIELD_INPUT_JS`, `SELECT_FILE_RADIO_JS`,
`LOAD_PRESET_JS`, кнопки по тексту `Add Task` / `Run`.

## 3. Итерация селекторов

При расхождении — снять свежий дамп проблемного экрана и сверить подписи/классы:
```
py aparser_monitor_ui.py --interactive     # копит HTML+PNG всех экранов в data/ui_dumps/
```
Прислать `manifest.jsonl` + нужный `dump_*.html`. Правки — точечно в JS-константах выше;
`CREATE_SELECTORS_READY` уже `True`.

## 4. Первое реальное создание (по-прежнему осторожно)

Когда превью корректно:
1. `autopilot_dry_run = false`, `autopilot_create_tasks = true`, `autopilot_max_new_tasks = 1`.
2. Один ручной прогон:
   ```
   py aparser_monitor_ui.py --autopilot --debug
   ```
3. Проверить: в Tasks Queue появилось **одно** задание под набор; в Telegram пришло
   `🤖 автопилот создал задания (1)`; autosend/stats потом подхватят `results/<набор>/`.
4. Убедиться, что повторный `--autopilot` **не плодит дубль** того же набора
   (идемпотентность: `_task_exists_for` по заголовку + `results/<набор>/`).

## 5. Ввод в расписание

- Отдельная задача планировщика `--autopilot`, реже монитора (15–30 мин).
- `autopilot_max_new_tasks` держать небольшим (1–2), чтобы не завалить очередь.
- Следить за логом: строки `autopilot: создано задание под набор …` / ошибки.

## 6. Откат

- Быстрый стоп: `autopilot_create_tasks = false` (вернётся Phase 1 — только уведомление).
- Полный тормоз: снять задачу `--autopilot` из планировщика.
- Ошибочно созданные задания удаляются из Tasks Queue вручную.

## Открытые вопросы к живой проверке

- Как именно A-Parser именует задание при нескольких файлах в наборе (влияет на
  идемпотентность по заголовку карточки) — уточнить по факту и при нужде переключить
  детект на имя результата.
- Грузит ли выбор Task preset настройки сам, или нужна кнопка загрузки (`LOAD_PRESET_JS`).
- Принимает ли Select File список через запятую как единый источник (в дампах — да).
